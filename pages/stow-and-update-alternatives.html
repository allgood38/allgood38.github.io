Stow and update alternatives

update-alternatives --install "/usr/bin/vim" editor "/usr/local/bin/vim" 100

Vim is actually stored at 

/usr/local/stow/bin/vim

And was installed using

cd /opt/vim
./configure ... --prefix=/usr/local/stow/vim
make && sudo make install
cd /usr/local/stow
sudo stow vim

But because of the update alternatives command, there is also a symlink installed in

/usr/bin/vim

And thanks to the update alternatives command, the bash $EDITOR variable is now pointing to vim, but should it stop working, it will automatically change it to something else, like nano.
